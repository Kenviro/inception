# Developer Documentation - Inception

## 1. Environment Setup
### Prerequisites
To build and run this project, the following tools must be installed on the host machine (Debian/Alpine VM):
- **Docker**: To manage containers.
- **Docker Compose**: To orchestrate the multi-container application.
- **GNU Make**: To execute the automation scripts provided in the Makefile.

### Configuration Files
The project structure is organized as follows:
- `srcs/`: Contains the `docker-compose.yml` and the environment file.
- `srcs/requirements/`: Contains the dedicated folders for each service (NGINX, MariaDB, WordPress) with their respective `Dockerfile` and configuration files.

### Secrets and Environment Variables
A `.env` file must be created in the `srcs/` directory before launching the project. It must contain:
- Database credentials (root password, user, password, database name).
- WordPress configuration (admin credentials, user credentials, domain name).

## 2. Build and Launch
The project uses a **Makefile** to simplify the deployment process.

* **Full Build**: `make` or `make up`
    1. Creates necessary directories in `/home/ktintim-/data/`.
    2. Builds images based on the custom Dockerfiles.
    3. Starts the containers in the private `inception` network.

* **Rebuild**: `make re`
    - Performs a full cleanup and restarts the build process from scratch.

## 3. Container and Volume Management
### Useful Commands
- **View running containers**: `docker ps`
- **Check container logs**: `docker logs <container_name>`
- **Access a container's shell**: `docker exec -it <container_name> /bin/bash` (or `sh` for Alpine).
- **List networks**: `docker network ls`
- **List volumes**: `docker volume ls`

### Troubleshooting
If a service fails to start:
1. Check the logs: `docker logs wordpress` or `docker logs mariadb`.
2. Ensure the port 443 is not already in use by another process on the host.
3. Verify that the `.env` file contains all required variables.

## 4. Data Persistence
All data generated by the services is stored outside the containers to ensure persistence even after a restart or deletion.

- **WordPress Files**: Located at `/home/ktintim-/data/wordpress`.
- **Database Files**: Located at `/home/ktintim-/data/mariadb`.

These paths are mounted as **Named Volumes** using the local driver with a bind mount configuration, ensuring compliance with the project's strict storage requirements.
